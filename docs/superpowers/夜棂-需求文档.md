## 一、产品概述

### 1.1 产品定位

**产品名称：** 夜棂

**一句话定位：**
把醒来就忘的梦，变成能留住的文字。

**最终愿景：**
每个人的梦里都住着另一个自己，那个自己比醒着的更真实、更勇敢、更脆弱。夜棂把那部分你写下来，像银行存款一样存起来，一年、十年后你还能翻回去看看那个在不同阶段住在你梦里的自己。

**slogan：**
你在夜棂存的每一篇梦，都是你自己。

**目标用户：**
- 18-35岁，对梦境好奇的人
- 有创作欲望，想把梦写成故事的人
- 失眠/睡眠不好，想记录梦的人

**核心价值主张：**

| 用户痛点 | 我们的解决方案 |
|---------|--------------|
| 醒来就忘80% | 引导用户第一时间记录 |
| 自己写不像 | AI补全逻辑和细节 |
| 梦太碎片不好看 | 重构为完整有画面感的故事 |
| 没有地方保存 | 自动归档，可随时查看 |

## 二、MVP功能定义

### 2.1 核心流程

```
用户醒来
↓
发送梦境碎片（语音/文字）
↓
AI追问细节（引导式提问）
↓
AI生成完整故事（500-800字）
↓
故事发送用户
↓
用户存档/分享/续写
```

### 2.2 功能范围（MVP）

**必须功能（P0）：**
1. 微信接收用户消息（语音/文字）
2. 碎片记录和存储
3. AI追问（标准话术）
4. 故事生成
5. 故事发送

**暂不做：**
- 小程序/App（先用微信跑通）
- 支付系统（先用免费跑通）
- 会员体系（等用户量起来再上）
- 插图生成（等流程跑通再扩展）

**用户A：晓琳，24岁，设计师**
```
睡前刷手机，醒来还记得梦的片段
但刷牙后就忘了一大半
想要记录下来但自己写不好

**用户B：大雄，32岁，产品经理**
```

```

**用户C：阿紫，28岁，写手**
```
对梦境写作有兴趣
想收集不同类型的梦作为创作素材
愿意分享自己的梦
```

### 3.2 用户故事地图

| 场景 | 用户行为 | 产品响应 |
|------|---------|---------|
| 早上醒来 | 用户记得做了一个梦 | 提醒文案触发 |

| 收到故事 | 用户阅读 | 自然续费引导 |
| 分享传播 | 用户觉得故事好 | 提供分享素材 |

## 四、功能详细设计

### 4.1 碎片接收

**输入方式：**
- 文字消息（用户直接发送）
- 语音消息（AI转文字后处理）

**接收标准：**
- 至少包含：场景描述 OR 人物 OR 情绪
- 不足时触发追问

**错误处理：**
- 用户发"没做梦" → 记录为空白，下次提醒
- 用户发送无关内容 → 友好提示

---

### 4.2 追问机制

**追问标准话术：**

```
收到你的碎片了

帮你追问几个细节，让故事更完整：

1. 梦里是白天还是晚上？
2. 有没有特别的东西出现？（物品/颜色/声音）
3. 叫你的人是熟悉的声音吗？
4. 醒来时是什么感觉？（害怕/难过/奇怪）
5. 梦里有没有特别的东西出现？

回答完帮你写成完整故事
```

**追问原则：**
- 最多追问3轮
- 每轮最多5个问题
- 引导而非审问
- 语气温和有画面感

---

### 4.3 故事生成

**生成标准：**

| 维度 | 要求 |
|------|------|
| 字数 | 500-800字（会员可解锁1500-3000字） |
| 风格 | 保留梦的荒诞感，不强行修复逻辑 |
| 叙事 | 有画面感，情绪连贯 |
| 结尾 | 可开放，不直接说教 |
| 视角 | 第一人称或第三人称视素材决定 |

**生成后处理：**
- 自动存档
- 生成时间戳
- 推送用户

---

### 4.4 故事交付

**交付话术：**

```
你的梦写完了

[故事全文]

---

写完了，你看看感觉对不对

这个梦里[一句话解读]

你的梦我帮你存档了
以后有新的梦可以继续发给我
```

**解读话术示例：**
- "梦里那种'追不过去'的感觉很多人都有"
- "红色在这个梦里可能代表某种被压抑的情绪"
- "这种醒来很冷的感觉，可能和梦里的孤独感有关"

## 五、变现设计

### 5.1 冷启动策略

**第一阶段：免费引流**
- 前100个梦免费写
- 不限次数，用户任意发
- 收集真实案例

**目标：**
- 跑通全流程
- 收集100个真实故事
- 打磨提示词和话术
- 建立初始口碑

---

### 5.2 会员体系（第二阶段）

**定价：**

| 类型 | 价格 | 权益 |
|------|------|------|
| 月卡 | 49元/月 | 无限次记录 + 故事生成（500-800字） |
| 年卡 | 399元/年 | 无限次记录 + 故事生成 + 续写功能 + 1500-3000字长篇 |
| 高级月卡 | 99元/月 | 无限次记录 + 无限次解读 + 续写 + 插图 |
| 高级年卡 | 899元/年 | 同上 + 优先体验新功能 |

**解读单次购买：** 9.9元/次

---

### 5.3 追销话术

**场景：故事交付后**

```
对了

我这边有个月卡
49元一个月 不限次数写梦

如果以后经常有梦想记录
或者想让我帮你续写这个故事
可以考慮一下

不强制的 觉得有用再说
```

## 七、内容规范

**版本A（痛点型）：**
```
你昨晚做了什么梦？

醒来就忘了一大半？

我做了一个小工具
帮你把梦的碎片写成完整故事

加我微信 帮你记录

**版本B（故事型）：**
```
我妈在梦里叫我
但我找不到她

梦到了一个很奇怪的地方
有雨，有楼梯，有一双红色的鞋子
我妈的声音一直在远处响
但我走不过去

躺了十分钟
细节已经模糊了一半

帮你把这种醒来就忘的梦

你梦到什么了？
发给我试试
```

## 九、风险与应对

| 风险 | 应对方案 |
|------|---------|
| 微信号被封 | 多账号 + 企业微信 |
| AI故事质量不稳定 | 先用提示词模板约束，收集反馈迭代 |
| 用户新鲜感消退 | 续写功能 + 社区 + 插图 |
| 竞争对手模仿 | 快速迭代 + 品牌壁垒 |
| 梦境素材涉及隐私 | 匿名化处理 + 用户授权机制 |
