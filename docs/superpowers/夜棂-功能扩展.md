# 夜棂 · 功能扩展文档

> 记录所有想到的产品功能扩展方向，供后期开发参考。
> 优先级排序不代表必须全部实现，根据实际运营情况迭代。

---

## 一、已确认的核心功能

### 1.1 梦境记录（核心功能）

- 用户通过语音或文字发送梦境碎片
- AI 追问细节（引导式提问）
- 生成完整故事（1500-3000字）
- 保留梦的荒诞感和情绪张力
- 支持多种风格：现实主义/悬疑惊悚/诗意隐喻/荒诞寓言

### 1.2 梦境存档

- 用户历史梦境故事保存
- 可按时间查看
- 可导出为 PDF/文本

## 二、功能扩展

### 2.1 记忆消失紧迫感

**功能描述：**
早上固定时间提醒用户记录梦境，解决"醒来就忘"的时间窗口问题。

**具体方案：**
- 早上8点自动推送消息："醒了吗？梦还记得多少？"
- 文案引导："刚醒的前10分钟是记录的黄金时间"
- 可自定义提醒时间

**优先级：** P1（高）
**价值：** 提高用户记录率，降低流失

---

### 2.2 故事交付仪式感

**功能描述：**
故事不只是文字，增加视觉和感官体验。

**具体方案：**

**a) 梦境插图**
- AI 根据故事关键词生成插图（一张）
- 风格：抽象、水墨、电影感
- 展示在故事顶部

**b) 语音朗读**
- TTS 语音生成，用户可闭眼"听"自己的梦
- 多种音色可选（温暖男声/温柔女声/旁白感）

**c) 梦境卡片**
- 故事精华 + 插图 生成一张可分享的图片
- 用户可一键发朋友圈
- 文案模板："我在夜棂记录了一个梦"

**优先级：** P2（中）
**价值：** 提升分享率和传播感

---

### 2.3 梦境解读（增值服务）

**功能描述：**
在故事基础上，增加心理层面的分析解读。

**具体方案：**

```
收到故事后，问用户：
"要不要听听这个梦可能代表什么？"

AI 分析：
1. 梦中出现的人/物/场景的象征含义
2. 情绪线索（你最近是不是压力很大？）
3. 一个开放性问题，引导自我思考

示例：
"梦里追不过去的感觉，可能和现实中某种'想要但得不到'的状态有关"
"那双红色的鞋子，可能是某种被压抑的欲望"
```

**优先级：** P1（高）
**价值：** 增加付费价值感，区分免费/付费功能

---

### 2.4 续写功能

**功能描述：**
用户可以继续扩展自己的梦境故事。

**具体方案：**

```
用户选项：
- "我想让故事继续往下走"
- "如果当时走过去了会怎样"
- "换一种结局"

AI 根据用户选择续写
保持原故事的氛围和元素
```

**优先级：** P2（中）
**价值：** 提升用户粘性和互动率

---

### 2.5 社区分享（梦墙）

**功能描述：**
用户匿名发布故事，其他用户可点赞评论。

**具体方案：**

```
"梦墙"功能：
- 用户可选择匿名发布故事
- 其他用户可点赞、评论
- 每周精选"最像电影的梦"
- 评论区的互动本身也是内容

运营机制：
- 每周选出10个精彩故事
- 入选故事的用户获得月卡优惠券
- 每月汇总"本月最受欢迎梦境"
```

**优先级：** P3（后期）
**价值：** 传播裂变，自然拉新

---

### 2.6 会员体系

**功能描述：**
分级会员制，不同权益对应不同价格。

**具体方案：**

| 会员类型 | 价格 | 权益 |
|---------|------|------|
| 普通会员（年卡） | 399元/年 | 无限次记录 + 基础故事生成 |
| 高级会员 | 99元/月 | 无限次记录 + 无限次解读 + 续写功能 + 每月一张插图 |
| 高级会员（年卡） | 899元/年 | 同上 + 优先体验新功能 |

**说明：**
- 高级会员的"解读"功能可单独购买（9.9元/次）
- 首批用户给予优惠价格

**优先级：** P1（高）
**价值：** 明确商业模式

---

### 2.7 实物延伸（可选）

**功能描述：**
延伸实体产品，增加收入来源。

**具体方案：**

| 产品 | 价格 | 场景 |
|------|------|------|
| 梦境笔记本 | 39元 | 线下场景，配合记录习惯养成 |
| 一年12篇故事合订本 | 99元 | 年末送礼 |
| 定制梦境插画装裱 | 199元 | 高端用户 |

**优先级：** P3（后期）
**价值：** 高客单价补充收入

## 三、运营功能

### 3.1 冷启动机制

**前100个梦 = 内容素材 = 案例展示**

```
运营策略：
1. 早期用户免费写（不限量）
2. 收集100个真实故事
3. 匿名发布"梦墙"作为产品展示
4. 精选10个精彩故事做成案例集
5. 作为文案素材反复使用
```

**优先级：** P0（最高）
**价值：** 冷启动和口碑积累

---

### 3.2 分享裂变

**功能描述：**
用户分享故事时带来新用户。

**具体方案：**

```
分享机制：
- 用户分享故事到朋友圈
- 朋友通过链接注册，双方获得月卡优惠券
- 邀请1个好友 = 延长1个月会期

传播素材：
- "梦境卡片"一键生成
- 精选故事匿名展示
```

**优先级：** P2（中）
**价值：** 降低获客成本

---

### 3.3 数据分析

**追踪指标：**

| 指标 | 说明 | 目标 |
|------|------|------|
| 次日留存率 | 第二天回访比例 | > 40% |
| 7日留存率 | 第七天回访比例 | > 20% |
| 故事生成成功率 | 用户发出碎片后成功拿到故事的比例 | > 95% |
| 续费率 | 会员到期后续费 | > 30% |
| 分享率 | 主动分享故事的用户比例 | > 10% |

**优先级：** P1（高）
**价值：** 数据驱动迭代

## 四、技术扩展

### 4.1 语音识别

- 内置录音 → 自动转文字
- 减少用户输入成本
- 适配"刚醒来懒得打字"的场景

### 4.2 AI 插图生成

- 根据故事关键词生成配图
- 风格可选（水墨/抽象/电影感）
- 建议用腾讯云CI或Stable Diffusion

### 4.3 定时提醒

- 用户设置每日提醒时间
- 早上/中午/任意时间
- 推送文案可自定义

### 4.4 多端同步

- 微信小程序
- App（iOS/Android）
- 网页版
- 数据云端同步
